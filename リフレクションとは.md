# リフレクションとは
Javaにおいて、クラス、フィールドなどの名称を指定するだけでアクセスできる機能。  

- Classインスタンス
```
Class<?> sampleClass = Class.forName("com.example.Sample");
```

- Constructorインスタンス
```
Constructor<?> constructor = sampleClass.getConstructor(int.class);
Object sampleInstance = constructor.newInstance(100);
```

- Fieldインスタンス
```
Field valueField = sampleClass.getDeclaredField("value");//取得  
Object value = valueField.get(sampleInstance); //値の取得
valueField.set(sampleInstance, 200); //値のセット
```

- Methodインスタンス
```
Method multiMethod = sampleClass.getMethod("multi", int.class); //取得 
Object ret = multiMethod.invoke(sampleInstance, 2); //実行
```

## 注意点
- セキュリティの問題  
悪意を持つクラスを差し込み、実行したいクラス名やメソッド名を指定するだけで実行できてしまう。  
- 保守性の低下   
他のクラスのメソッド名、フィールド名を過度に指定することは、「知りすぎている状態」にあたる。  
メソッド名やフィールド名を変更したいとき、変更箇所が増える。  
- 名称に誤字が含まれていてもコンパイルエラーにならない
そこが原因でバグや例外が発生したとき、デバッグが難しくなる。

## いつ使うのか
通常のシステム開発で使われることは少ないが、フレームワークを開発する際などにはよく使われる。  
どのクラス、どのフィールド、どのメソッドかは具体的にはわからないが操作したい、というシーンが多いため。
たとえば、以下はSpring FrameworkのAutowiredAnnotationBeanPostProcessor.javaの一部。  

以下のようなコードがあったとき、
```
class OrderService(
    @Autowired val userService: UserService
)
```
@AutowiredがついているUserServiceクラスのBeanを取得し、OrderServiceクラスのフィールドとしてセットする処理。
```
@Override
		protected void inject(Object bean, @Nullable String beanName, @Nullable PropertyValues pvs) throws Throwable {
			Field field = (Field) this.member;
			Object value;
			if (this.cached) {
				try {
					value = resolveCachedArgument(beanName, this.cachedFieldValue);
				}
				catch (BeansException ex) {
					// Unexpected target bean mismatch for cached argument -> re-resolve
					this.cached = false;
					logger.debug("Failed to resolve cached argument", ex);
					value = resolveFieldValue(field, bean, beanName);
				}
			}
			else {
				value = resolveFieldValue(field, bean, beanName);
			}
			if (value != null) {
				ReflectionUtils.makeAccessible(field);
				field.set(bean, value); //ここ
			}
		}
```
出典：https://github.com/spring-projects/spring-framework/blob/main/spring-beans/src/main/java/org/springframework/beans/factory/annotation/AutowiredAnnotationBeanPostProcessor.java  

## なぜFlutterでは使用禁止なのか？  
Dartにもリフレクションと似た機能はあるが、Flutter上では使用不可。
これはDart・Flutterの思想と相性が悪いから。   
Dartは動作の高速性を担保するために、Cと同様にコンパイル時に機械語に落とすようになっている。  
その際、使用しない処理はツリーシェイキングで削除するが、リフレクションはコンパイル時には使われるかどうか判断できない。  
さらに、クラス名、フィールド名などの情報を文字列で保持して名前ベースで探すより、直接そのアドレスに飛んだほうが効率が良い。  
一方、JavaはいったんJVM上で動くバイトコードに変換され、実行時に機械語になる。  
そのため、実行時にチューニングする余地が大きい。
