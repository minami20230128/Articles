# 移譲の隠蔽  
## 概要  
サーバオブジェクト（後述のDepartmentクラスのように間に挟まっているもの）を通して移譲先（後述のmanager）のメソッドを実行するとき、  
クライアント（呼び元）は移譲先のメソッド名を知っていなければならない。  
これを解消するため、サーバオブジェクトに移譲先のメソッドを隠蔽するメソッドを設ける。

## 例 
```class Person  
constructor(name) {
    this._name = name;
}

get name() { return this._name; }
get department() { return this._department; }
set department(arg) { this._department = arg; }
```

```class Department
get changeCode() { return this._changeCode; }
set changeCode(arg) { this.changeCode = arg; }
get manager() { return this._manager; }
set manager(arg) { this._manager = arg; }
```

ある人のマネージャーの情報を取得するとき、以下のようにする必要がある。  
```
manager = ePerson.department.manager;
```
Personクラスに
```
get manager() { return this._department.manager; }
```
を設けると、
```
manager = ePerson.manager;
```
だけで同じ情報が取得できるようになる。
