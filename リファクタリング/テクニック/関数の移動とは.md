# 関数の移動とは
## 例1 入れ子関数をトップレベル関数に変更  
以下のcalculateDistance()を、trackSummary()に依存せずに動くようにしたい。  
```
function trackSummary(points) {
    const totalTime = calculateTime();
    const totalDistance = calculateDistance();
    const pace = totalTime / 60 /totalDistance;

    return {
        time: totalTime,
        distance: totalDistance,
        pace: pace
    }

    function calculateDistance() {
        let result = 0;
        for (let i = 1; i < points.length; i++) {
            result += distance(points[i - 1], points[i]);
        }
        return result;

        function distance() {}
        function radians() {}
        function calculateTime() {}
    }
}
```

移動前の関数 = calculateDistance()  
移動後の関数 = top_calculateDistance()  

1. いきなり移動前の関数を消さず、trackSummary()の外側にコピペ。仮の名前をつける（top_〇〇など）。
2. 依存している関数・変数の処理  
calculateDistance()はpointとdistance()に依存している。  
- pointsはメンバ変数なのでパラメータとして渡すようにする。  
- distance()はcalculateDistance()の中の入れ子関数。
（distance()の詳細は書籍にて）distance()はradians()のみに依存し、radians()は何にも依存していないので一緒に移動
3. 静的解析を行う。
lintなどで未定義シンボルが残っていないか確認する。
4. 移動後の関数を移動前の関数から呼び出す形式で実行し、問題なく動作することを確認する。
5. 移動前の関数を消し、もともと呼ばれていた箇所を移動後の関数で置き換える。
