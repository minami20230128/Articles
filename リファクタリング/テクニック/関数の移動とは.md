# 関数の移動とは
## 例1 入れ子関数をトップレベル関数に変更  
以下のcalculateDistance()を、trackSummary()に依存せずに動くようにしたい。  
```
function trackSummary(points) {
    const totalTime = calculateTime();
    const totalDistance = calculateDistance();
    const pace = totalTime / 60 /totalDistance;

    return {
        time: totalTime,
        distance: totalDistance,
        pace: pace
    }

    function calculateDistance() {
        let result = 0;
        for (let i = 1; i < points.length; i++) {
            result += distance(points[i - 1], points[i]);
        }
        return result;

        function distance() {}
        function radians() {}
        function calculateTime() {}
    }
}
```

移動前の関数 = calculateDistance()  
移動後の関数 = top_calculateDistance()  

1. いきなり移動前の関数を消さず、trackSummary()の外側にコピペ。仮の名前をつける（top_〇〇など）。
2. 依存している関数・変数の処理  
calculateDistance()はpointとdistance()に依存している。  
- pointsはメンバ変数なのでパラメータとして渡すようにする。  
- distance()はcalculateDistance()の中の入れ子関数。
（distance()の詳細は書籍にて）distance()はradians()のみに依存し、radians()は何にも依存していないので一緒に移動
3. 静的解析を行う。
lintなどで未定義シンボルが残っていないか確認する。
4. 移動後の関数を移動前の関数から呼び出す形式で実行し、問題なく動作することを確認する。
5. 移動前の関数を消し、もともと呼ばれていた箇所を移動後の関数で置き換える。
6. 関数名を再考してもよいかも？

## 例2 クラス間での移動  
``` class Account
get bankCharge(){
    let result = 4.5;
    if (this._daysOverdrawn > 0) result += this.overdraftCharge;
    return result;
}

get overdraftCharge() {
    if (this.type.isPremium) {
        const baseCharge = 10;
        if (this.daysOverdrawn <= 7)
            return baseCharge;
        else
            return baseCharge + (this.daysOverdrawn - 7) * 0.85;
    }
    else
        return this.daysOverdrawn * 1.75;
}
```

口座の種類ごとに手数料計算アルゴリズムを切り替えるような変更が入る可能性があるので、overdraftCharge()をAccountTypeクラスに移動したい。  
1. overdraftCharge()が依存するメソッドや属性をまとめて移動するか検討  
isPremiumはtypeのメンバーなので「this.」で呼び出せる。  
daysOverdrawnは値として渡す方法と、Accountで渡す方法が考えられ、将来Accountクラスの別の情報が必要になったら後者に変更する可能性はあるが、まず値として渡すことにする。  
2. 呼び出し元を移譲呼び出しに置き換える  
```
get overdraftCharge() {
    return this.type.oberdraftCharge(this.daysOverdrawn);
}
```
