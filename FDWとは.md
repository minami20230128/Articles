# FDWとは  
PostgreSQLで、データベースから外部データソースを参照できるようにするための機能。  
もともと外部DBと通信できるdblinkという機能があったが、dblinkでの通信ではモジュール関数を使うしかないのに対して、  
FWDだとSQLでデータ取得できる。

## どういうときに使うのか  
たとえば複数のDBで共通して使うデータがあるとき。  
各DBに同じテーブルを設けていては容量の無駄になってしまうので、独立したDBにテーブルを設け、FWDで通信するようにしたほうが効率的。  

参考：  
https://techblog.goinc.jp/entry/2021/12/15/090000_1

## セットアップ手順
1. 接続先DB・接続元DBの両方でプラグインを有効化する
```
CREATE EXTENSION postgres_fdw;
```

2. 接続先DBの設定を行う  
外部サーバ名には、接続先DBを表す名前を入れる。
```
CREATE SERVER ${外部サーバ名} FOREIGN DATA WRAPPER postgres_fdw
OPTIONS (
    host '${接続先ホスト名}',
    dbname '${接続先データベース名}',
    port '${接続先データベースのポート番号}'
);
```

3. 接続元DBで、接続するためのユーザを作成する
```
CREATE USER MAPPING FOR ${接続元データベースのユーザー名} SERVER ${外部サーバ名}
OPTIONS (
    user '${接続先データベースのユーザー名}',
    password '${接続先データベースのユーザーパスワード}'
);
```

4. 外部テーブルの定義  
接続元DBに、接続先DBのテーブルを表す外部テーブルを作成する。  
以下のSQLだと、テーブルを一つ一つ作るのではなくスキーマごと作成できる。
```
IMPORT FOREIGN SCHEMA ${接続先データベースのスキーマ名}
FROM SERVER ${外部サーバ名}
INTO ${接続元データベースに配置するスキーマ名};
```
