# インサイダー取引とは
一方のクラスがもう一方のクラスの内部事情を知りすぎていること。
```
class Customer:
    def __init__(self, name, status, discount_rate):
        self.name = name
        self.status = status
        self.discount_rate = discount_rate


class Order:
    def __init__(self, customer, amount):
        self.customer = customer
        self.amount = amount

    def final_price(self):
        if self.customer.status == "vip":
            return self.amount * (1 - self.customer.discount_rate)
        return self.amount
```
このコード上では、Orderクラスの中でself.amount、self.customer.discount_rateといったCustomerクラスのメンバを呼び出しまくっている。  
結果として、Customerクラスの変更を行うとorderクラスの変更まで必要になってしまう場合がある。

以下のように、Customerクラスで割引計算を行うようにすれば、OrderクラスはCustomerクラスの中身を知らなくて良くなる。
```
class Customer:
    def __init__(self, name, status, discount_rate):
        self.name = name
        self.status = status
        self.discount_rate = discount_rate

    def price_after_discount(self, base_amount):
        if self.status == "vip":
            return base_amount * (1 - self.discount_rate)
        return base_amount


class Order:
    def __init__(self, customer, amount):
        self.customer = customer
        self.amount = amount

    def final_price(self):
        return self.customer.price_after_discount(self.amount)
```

## 有効なテクニック
- 関数の移動
- フィールドの移動
- 移譲の隠蔽
「メンバ.そのプロパティ.更にそのプロパティ」といったように、深いネストでプロパティを呼び出しているとき、  
真ん中のプロパティのクラスにプロパティを返す関数を追加すること。
